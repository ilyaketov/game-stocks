# Стоки игр — облачная версия

Веб-приложение для построения отчётов по остаткам игровых ключей из выгрузок QuickBooks Inventory Report. Запускается на [Streamlit Community Cloud](https://share.streamlit.io/) — бесплатно, доступно из браузера, без установки.

## Особенности облачной версии

- **Сессионный режим:** данные не сохраняются между сессиями. Закрыл вкладку — всё стёрлось. Это особенность бесплатного хостинга Streamlit.
- **Использование:** загрузил файл(ы) → получил отчёты → скачал. Чтобы получить корректные «Дни с последнего закупа», нужно загрузить **всю историю** (одним многомесячным файлом или несколькими одномесячными).
- **Бесплатно** для одного пользователя, до 1ГБ RAM, можно сделать приватным (ограничение по email).

Для регулярной работы с накоплением истории есть **локальная версия** — она хранит файлы в папке на компьютере между запусками.

## Деплой на Streamlit Community Cloud

### Шаг 1: Создать репозиторий на GitHub

1. Зайдите на [github.com](https://github.com) и создайте новый репозиторий (можно публичный или приватный — для приватного нужен план GitHub Pro / Streamlit Cloud Pro, но для публичного бесплатно).
2. Распакуйте `stock_app_cloud.zip` локально.
3. Загрузите все файлы в репозиторий. Можно через веб-интерфейс GitHub (drag & drop), либо через `git`:
   ```bash
   cd stock_app_cloud
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<ваш-username>/<репо>.git
   git push -u origin main
   ```

### Шаг 2: Подключить к Streamlit Cloud

1. Зайдите на [share.streamlit.io](https://share.streamlit.io/) и войдите через GitHub.
2. Нажмите **«New app»**.
3. Выберите свой репозиторий, ветку `main`, и в поле **«Main file path»** укажите `app.py`.
4. (Опционально) В **«App URL»** задайте короткий URL вида `stocki-igr.streamlit.app`.
5. Нажмите **«Deploy»**. Через 2-5 минут приложение будет доступно по полученному URL.

### Шаг 3: Сделать приватным (опционально)

В настройках приложения на Streamlit Cloud:
- **Settings → Sharing → Allow only specific people**
- Добавьте email-адреса коллег, которые должны иметь доступ.

Доступ будет предоставлен через Google-аккаунт или email-логин.

## Как пользоваться

1. Откройте URL приложения в браузере.
2. Перетащите xlsx-файл(ы) в зону загрузки или выберите кнопкой.
3. Приложение автоматически распознает периоды в данных и сгенерирует отчёты на конец каждого месяца.
4. Скачайте нужные отчёты кнопками «Скачать xlsx».

**Поддерживаются:**
- Одномесячные выгрузки (например, `Stock_2026-03.xlsx`)
- Многомесячные выгрузки (`2025-12_-_2026-03.xlsx`)
- Несколько файлов сразу — они будут объединены, при пересечении периодов приоритет у файла с большим числом транзакций.

## Локальный запуск (для разработки)

```bash
pip install -r requirements.txt
streamlit run app.py
```

Откроется `http://localhost:8501`.

## Что в каждом отчёте

5 листов:

- **Источник** — копия исходной выгрузки за этот месяц
- **Свод** — итоги по подразделениям (на формулах SUMIF)
- **Таблица** — плоская таблица: одна строка = пара (артикул × магазин), с подсветкой по дням и жирным шрифтом для позиций без движения
- **Отчёт** — то же, но сгруппировано по магазинам с подытогами
- **Риски** — анализ стареющего стока (3 категории), Топ-15 «мёртвых» позиций, выводы

## Логика расчётов

- **Остаток на конец** = `SUM(Qty)` по транзакциям пары до конца периода (а не последнее `Qty on hand` — эта колонка часто содержит ошибки в выгрузках).
- **Остаток на начало** = `end - bought + sold - adj_in + adj_out` (баланс самосогласован).
- **Дней с последнего закупа** = разница между концом периода и датой последнего `Bill` для этого артикула в любом магазине, по всей загруженной истории.
- **Перемещения между магазинами** не отражаются в продажах/закупках, но участвуют в расчёте остатка на начало.
- **Inventory Starting Value** игнорируется (служебная нулевая транзакция).
